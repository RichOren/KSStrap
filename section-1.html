1 Buttons

Buttons have a few required constraints for proper functionality, accessibility, and styling.

  • Whenever possible, use <button type="button"> over <a href="#">.
  • Always declare a type on your <button> elements to ensure zero hiccups with nearby forms and property order.
  • Never set tabindex on buttons—let the browser automatically set that.
  • For tooltips, always place the aria-label and .tooltipped classes directly on the <button> or <a>.
  • Icon and tooltips conflict, therefore Icons should always be children elements of <button>s or <a>s.

1.1 Base styles & states

These are the base styles and states.

:hover

Buttons will appear rolled over (with a darker background, darker border) when the pointer device enters this elements box model on 'non touch' devices. This pseudo property has no effect on touch devices.

:active

Buttons will appear pressed (with a darker background, darker border, and inset shadow) when active. For <button> elements, this is done via :active. For <a> elements, it's done with .active. However, you may use .active on <button>s should you need to replicate the active state programmatically.

.active

No need to add :active as it's a pseudo-class, but if you need to force the same appearance, go ahead and add .active. Add the .active class to <a> buttons.

.disabled

Disabled State - Make buttons look unclickable by fading them back 50%. If you use <button> element, that is acceptable to add disabled="disabled" attribute instead. On Anchor element Add the .disabled class to <a> buttons.

1.2 Sizes

Fancy larger or smaller buttons? Add .btn-lg, .btn-sm, or .btn-xs for additional sizes. Create block level buttons—those that span the full width of a parent— by adding .btn-block. The default button size is shown below.

.btn-lg

The large button size is good for very large screen sizes and TV's. They are also the preferred size to use on Touch devices where the user will need a larger hit area on the screen.

.btn-sm

Small button.

.btn-xs

Mini button.

.btn-block

Block level button.

1.3 Button tags

Use the .btn class on an <a>, <button>, or <input> element.

Link 
<a href="#" class="btn btn-default">Link</a>
<button type="submit" class="btn btn-default">Button</button>
<input type="submit" class="btn btn-default" value="Input:submit" />
<input type="button" class="btn btn-default" value="Input:button" />

1.5 Button groups

Group a series of buttons together on a single line with the button group.

Tooltips & popovers in button groups require special setting

When using tooltips or popovers on elements within a .btn-group, you'll have to specify the option container: 'body' to avoid unwanted side effects (such as the element growing wider and/or losing its rounded corners when the tooltip or popover is triggered).

.btn-group

Wrap a series of buttons with .btn in .btn-group.

<div class="btn-group">
  <button type="button" class="btn btn-default">Left</button>
  <button type="button" class="btn btn-default">Middle</button>
  <button type="button" class="btn btn-default">Right</button>
</div>
